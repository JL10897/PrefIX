   1·    # Codex-Oriented Minimal Orchestrator Specification  
Two-Turn Conversation (User-Initiated, Stateless Agent)

This document describes the **simplest possible orchestrator** for connecting a user-simulator LLM and an agent-LLM inside a Jupyter Notebook.  
Codex should implement exactly this specification and **nothing more**.

The goal is:  
**Run two user→agent turns and output the interaction history.**

No preferences, no error-injection, no tool simulation, no validation, no multi-turn generalization.

---

# 1. Objective

Codex must produce a Jupyter Notebook that:

1. Defines two system prompts:
   - `user_system_prompt`
   - `agent_system_prompt`

2. Defines a task string `TASK`.

3. Implements **exactly two full user→agent cycles**:
   - Turn 1: user speaks, agent responds
   - Turn 2: user speaks, agent responds

4. For each LLM call, Codex must:
   - Inject the correct system prompt
   - Provide the full history so far
   - Append the model output to `history`

5. The final output is a JSON-style Python list:
   ```
   history = [
       {"role": "user", "content": "..."},
       {"role": "assistant", "content": "..."},
       {"role": "user", "content": "..."},
       {"role": "assistant", "content": "..."}
   ]
   ```

That is all.

---

# 2. Components to Implement in Notebook

Codex must structure the notebook with the following conceptual sections:

## 2.1 Imports
- Import the OpenAI client (or equivalent call wrapper).
- No advanced libraries required.

## 2.2 System Prompts
Notebook must define two variables:

```
user_system_prompt = """
<Describe user speaking style only. No task here.>
"""

agent_system_prompt = """
<Basic constraints, no persona, no task.>
"""
```

## 2.3 Task Definition
Notebook must define:

```
TASK = "..."
```

The user simulator receives this; the agent does not.

## 2.4 History Initialization
Notebook must define:

```
history = []
```

History is a Python list storing dictionaries of the form:

```
{"role": "user" or "assistant", "content": "..."}
```

Codex does not need to enforce deeper structure.

---

# 3. Message Construction Rules

## 3.1 User Input (Turn 1)

Codex must call the user LLM with:

- `system`: `user_system_prompt`
- `input`: task + empty history
- User instruction: begin the conversation naturally based on the task

The call should assemble something like:

```
messages = [
  {"role": "system", "content": user_system_prompt},
  {"role": "user", "content": f"Task: {TASK}
History: (empty)
Please start the conversation."}
]
```

## 3.2 Agent Input (All Turns)

Codex must call the agent LLM with:

- `system`: `agent_system_prompt`
- `input`: the full history so far
- Instruction: respond to the last user message

Example:

```
messages = [
  {"role": "system", "content": agent_system_prompt},
  {"role": "user", "content": format(history)}
]
```

## 3.3 User Input (Turn 2)

Same as Turn 1, except “History” now contains previous turns:

```
Task: <TASK>
History:
1. user: ...
2. assistant: ...
Please continue as the user.
```

---

# 4. Two-Turn Loop Specification

Codex must implement exactly the sequence below:

```
# Turn 1
user_output = call_user_with(task, empty_history)
append to history: {"role": "user", "content": user_output}

agent_output = call_agent_with(history)
append to history: {"role": "assistant", "content": agent_output}

# Turn 2
user_output = call_user_with(task, history)
append to history

agent_output = call_agent_with(history)
append to history

# stop here
```

### No loops beyond two cycles.  
### No tool execution.  
### No additional logic.

---

# 5. Final Output Format

Codex must print or return:

```
history
```

Example structure:

```
[
  {"role": "user", "content": "..."},
  {"role": "assistant", "content": "..."},
  {"role": "user", "content": "..."},
  {"role": "assistant", "content": "..."}
]
```

---

# 6. Notebook Structure Summary

**Cell 1:** Imports  
**Cell 2:** System prompts  
**Cell 3:** Task + history initialization  
**Cell 4:** Functions to call user and agent models  
**Cell 5:** Turn 1 execution  
**Cell 6:** Turn 2 execution  
**Cell 7:** Print history  

Codex should follow this structure closely.
